# Plan: Product Comparison Feature (Сравнение товаров)

## Overview
Add session-based product comparison — users can add products to a comparison list from product cards/detail pages, then view them side-by-side on a dedicated comparison page.

## Architecture
Follows the same pattern as the cart: session storage (`apps/cart/cart.py` style), JSON API endpoints, Vue.js frontend, Bulma CSS. No database models needed — comparison is session-only.

## Files to Create

### 1. `apps/comparison/comparison.py` — Comparison class
Session-based storage modeled after `apps/cart/cart.py`.
- Session key: `'comparison'`
- Stores: `{product_id: True}` (just presence tracking, no quantities)
- Methods: `add(product_id)`, `remove(product_id)`, `get_product_ids()`, `get_products()`, `__len__()`, `clear()`
- Max limit: 4 products
- `get_products()` returns Product queryset with `select_related('category__main_category', 'brand')` + `prefetch_related('variable_set__varitem')`

### 2. `apps/comparison/views.py` — Comparison page view
- `comparison_detail(request)` — renders comparison table with all product attributes + variables

### 3. `apps/comparison/context_processors.py` — Inject comparison count
- `comparison(request)` → `{'comparison': Comparison(request)}`

### 4. `apps/comparison/templates/comparison.html` — Comparison page
Bulma table layout showing products side-by-side:
- Product image + title (linked)
- Brand
- Price (with/without VAT)
- In-stock status
- SKU, Article
- Category, Main category
- Technical characteristics (variables) — rows for each unique variable across compared products
- Add to cart button per product
- Remove from comparison button per product

Uses Vue.js (`#comparisonapp`) with `[[ ]]` delimiters matching project conventions.

### 5. `apps/comparison/api.py` — JSON API endpoints
- `api_add_to_comparison(request)` — POST `{product_id}`, returns `{success, count}`
- `api_remove_from_comparison(request)` — POST `{product_id}`, returns `{success, count}`
- `api_toggle_comparison(request)` — POST `{product_id}`, returns `{success, count, added: bool}`

### 6. `apps/comparison/apps.py` — AppConfig

### 7. `apps/comparison/__init__.py`
